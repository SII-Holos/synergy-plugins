# Contributing Plugins

Synergy Plugin Market uses GitHub pull requests as the review and publishing channel.

Read [REVIEW_POLICY.md](REVIEW_POLICY.md) before submitting plugins that request high-risk permissions, `verified`, or `official` status.

## Before Opening a PR

- Publish the plugin artifact from the plugin's own repository.
- Use an installable `.synergy-plugin.tgz` generated by `synergy-plugin pack` or `synergy-plugin publish-market`.
- Sign the artifact and upload the `.sig` file next to it. The entry must record the signer public key from that `.sig` file.
- Use a stable canonical plugin id. The file name, registry id, detail entry id, tarball manifest `name`, and signature `pluginId` must all match.
- If the plugin has a custom marketplace icon, include `icons/<plugin-id>.svg` and set the entry `icon` to `{ "type": "registry-svg", "path": "icons/<plugin-id>.svg" }`. Simple entries may use `{ "type": "lucide", "name": "<token>" }`.
- Keep metadata factual. Do not list tools, UI surfaces, or permissions that are not present in the packaged artifact.

The easiest path from a plugin repository is:

```bash
synergy-plugin publish-market --repo https://github.com/owner/my-plugin
```

It prepares the release assets and registry PR. If GitHub upload, push, or PR creation cannot be automated, it prints the manual fallback commands.

## Local Validation

```bash
bun install
bun run build-registry
bun run validate
bun run build-registry --check
```

`registry.json` is generated from `plugins/*.json`. If validation reports drift, run:

```bash
bun run build-registry
```

and commit the generated index.

## Review Criteria

Maintainers check that:

- the plugin repository and release are public and inspectable
- the artifact is an installable package rather than a source archive
- permissions and risk are clearly represented
- integrity and signature metadata match the release artifact
- the registry signer matches the `.sig` signer and verifies the Ed25519 signature
- custom SVG icons are small, local, and free of scripts, event handlers, `foreignObject`, embedded images, and external references
- the plugin is compatible with the listed Synergy version range

Third-party submissions should not set `official: true` or `verified: true` unless a maintainer explicitly asks for that change.
